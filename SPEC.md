# 스펙 정의 문서

## 1. 소개

### 1.1 서비스 정의
링크에 접근할 수 있는 사용자 모두가 같은 플레이리스트를 공유할 수 있는 서비스.

### 1.2 서비스 목표
다양한 음악 플랫폼(유튜브, 사운드클라우드 등)에 분산된 음악을 하나의 리스트로 통합하여 공유할 수 있는 서비스를 제공한다.

## 2. 기능 요구사항

### 2.1 플레이리스트
- 플레이리스트란 여러 음악 항목을 모아 놓은 단위이다. 각 플레이리스트 안에는 음악의 제목, 아티스트, 접근 링크를 포함한 음악 목록이 표시된다.
- 메인 화면에서 플레이리스트를 추가(추가 버튼), 선택(클릭), 편집(편집 버튼)할 수 있다.
- 플레이리스트를 추가: 추가 화면에서, 플레이리스트의 제목과 썸네일을 입력해 새로운 플레이리스트를 추가할 수 있다. 저장 시 변경 사항은 즉시 화면에 반영된다.
- 플레이리스트 선택: 해당 플레이리스트의 링크를 연다.
- 플레이리스트 편집: '편집' 버튼을 누르면 편집 모드로 전환되며, 각 항목에 수정 및 삭제 버튼이 표시된다. '삭제' 버튼을 누르면 "해당 플레이리스트를 삭제하시겠습니까?" 라는 작은 팝업이 나온다. 그 화면 안의 '삭제' 버튼을 누르면 플레이리스트가 삭제된다. '수정' 버튼을 누르면 해당 플레이리스트의 수정 화면이 나온다. 플레이리스트의 제목과 썸네일을 수정할 수 있다. 저장 시 변경 사항은 즉시 화면에 반영된다.
- 필수 입력값이 비어 있을 경우 저장되지 않으며, 사용자에게 오류 메시지를 표시한다.
- 각 플레이리스트는 고유 링크를 가지며, 해당 링크에 접근한 사용자는 해당 플레이리스트를 조회하고 편집할 수 있다.
- 플레이리스트가 없을 경우 빈 상태 메시지와 추가 버튼을 표시한다.
- 플레이리스트는 다음 정보를 가진다:
  - playlist_id (고유값)
  - 제목 (필수)
  - 썸네일 (선택, 이미지 업로드를 통해 등록된 이미지)

### 2.2 음악
- 음악은 플레이리스트에 리스트업된 항목이다.
- 한 플레이리스트에 음악을 추가할 때, 오직 선택된 플레이리스트에만 추가된다.
- 각 플레이리스트 링크 안에서 음악을 추가, 편집할 수 있다. 플레이리스트 링크에 접근했을 때 나타나는 것은 음악 추가 화면이다.
- 음악 추가: 음악 추가 화면에서 음악을 추가할 수 있다. 음악을 추가할 때, 음악의 제목과 아티스트를 필수적으로 입력해야 한다. 접근 링크는 선택 사항이며, URL 형식이 아니어도 입력할 수 있다. 입력된 값은 텍스트로 표시된다. '저장' 버튼을 누르면 사용자가 입력한 음악이 플레이리스트에 추가된다. 저장 시 변경 사항은 즉시 화면에 반영된다.
- 음악 편집: '음악 편집 화면'에서 '편집' 버튼을 누르면 각 음악 옆에 '삭제' 버튼과 '수정' 버튼이 생긴다. '삭제' 버튼을 누르면 해당 음악이 삭제된다. '수정' 버튼을 누르면 해당 음악의 제목과 아티스트와 링크를 수정할 수 있다. 저장 시 변경 사항은 즉시 화면에 반영된다.
- 링크 유형 판별: 음악 링크가 입력된 경우, 시스템은 링크 유형을 자동 판별한다. 판별 유형은 YouTube, Apple Music, Soundcloud, link 이다.
- 링크 유형 표시: 링크가 있는 음악 항목은 링크 텍스트를 링크 유형 라벨(YouTube, Apple Music, Soundcloud, link)로 표시한다.
- 플랫폼 필터: 사용자는 플랫폼 선택 버튼(모두 보기, YouTube, Apple Music, Soundcloud, link)으로 음악 목록을 필터링할 수 있다. 특정 플랫폼을 선택하면 해당 링크 유형의 음악만 목록에 표시된다.
- 목록 복사 범위: 음악 목록 복사 기능은 현재 화면에 표시된 목록만 복사한다. 즉, 검색어 및 플랫폼 필터가 적용된 결과만 복사 대상이다.
- 링크 예외 처리: 링크가 URL 형식이 아니거나 판별 규칙에 맞지 않으면 link 유형으로 처리한다.
- 필수 입력값이 비어 있을 경우 저장되지 않으며, 사용자에게 오류 메시지를 표시한다.
- 음악 추가 화면에 음악이 없을 경우 해당 플레이리스트에 음악이 없음을 안내하는 메시지를 표시한다.
- 음악은 다음 정보를 가진다:
  - music_id
  - title
  - artist
  - link (optional)
  - playlist_id

### 2.3 Apple Music 내보내기 (현재 구현)
- 사용자는 플레이리스트 상세 화면에서 Apple Music 내보내기 모달을 열 수 있다.
- 내보내기 대상은 Apple Music 링크로 판별된 곡만 포함한다.
- 현재 구현은 Apple Music API 인증 기반의 직접 생성 방식이 아니라, 곡별 Apple Music 링크를 열어 사용자가 직접 플레이리스트에 추가하는 방식이다.
- 내보내기 모달에는 곡 제목/아티스트와 함께 Apple Music 열기 버튼을 제공한다.
- Apple Music 링크가 없는 경우 사용자에게 오류 메시지를 표시한다.

### 2.4 장르 메타데이터 및 시각화 (구현)
- Apple Music/iTunes 검색으로 음악을 추가할 때, iTunes Search API의 `primaryGenreName` 값을 함께 저장한다.
- iTunes Search API의 곡 단위 장르 정보는 단일 대표 장르를 기준으로 처리한다.
- 음악 목록에서 장르 정보가 있는 경우, 아티스트 정보 옆에 장르 라벨을 함께 표시한다.
- 장르 라벨은 아티스트 텍스트와 시각적으로 구분되어야 하며, 과도하게 튀지 않는 보조 정보 스타일을 사용한다.
- 플레이리스트 상세 화면에서 플레이리스트 썸네일 근처에 원형(도넛) 차트를 표시한다.
- 도넛 차트는 Apple Music/iTunes 검색으로 추가된 곡 중 장르 정보가 있는 항목만 집계 대상으로 사용한다.
- 차트는 장르별 곡 수 비율을 표시하며, 비율 합은 100% 기준으로 계산한다.
- 장르 정보가 없는 곡은 기본적으로 집계에서 제외하며, 필요 시 별도 Unknown 범주로 확장할 수 있다.
- 장르명이 긴 경우, 데스크톱에서는 가급적 원문 표기를 유지하고 모바일에서만 제한적으로 축약/말줄임을 적용한다.
- 기존 데이터 보강을 위해, 장르가 비어 있고 Apple Music/iTunes 링크에서 곡 ID를 추출할 수 있는 항목은 iTunes lookup API를 통해 장르를 자동 백필한다.

### 2.5 앨범 커버 메타데이터 및 표시 (구현)
- iTunes 검색으로 음악을 추가할 때 `artworkUrl100`(fallback: 60/30)을 `artwork_url`로 저장한다.
- 기존 데이터 중 `artwork_url`이 비어 있고 링크에서 곡 ID를 추출할 수 있는 항목은 iTunes lookup API를 통해 커버 URL을 자동 백필한다.
- 음악 목록의 앨범 커버는 데스크톱 뷰(lg 이상)에서만 표시한다.
- 커버가 없는 항목은 랜덤 도형 플레이스홀더를 표시한다.
- 플레이스홀더는 5개 패턴(4개 코너 부채꼴 + 1개 원)을 동일 확률로 선택하며, 곡 단위로 고정된 시드 기반 랜덤을 사용한다.
- 플레이스홀더 보조색은 디자인 테마의 그래픽 보조색 `#D4A84F`를 사용한다.

### 2.6 공유
- 사이트 링크에 접근할 수 있는 모든 사용자는 모든 플레이리스트를 조회하고 편집할 수 있다.
- 플레이리스트 링크에 접근할 수 있는 모든 사용자는 모든 플레이리스트를 조회하고 편집할 수 있다.

#### 디자인 테마 (현재 구현 기준)

**테마 방향**
- 따뜻한 뉴트럴 베이스(베이지/그레이) 위에 포인트 레드 계열을 사용하는 미니멀 카드 UI
- 메인(플레이리스트 목록)은 Figma 기준 톤으로 반영되었고, 상세 페이지는 기존 스타일이 일부 유지됨

**타이포그래피**
- 기본 폰트: Space Grotesk
- 한글 fallback: Noto Sans KR, Apple SD Gothic Neo
- 메인 타이틀("Mayonnaise"): 4rem, weight 700, tracking -0.02em
- 메인 서브타이틀: 1.5rem, weight 400
- 카드 타이틀: 1.75rem, weight 700
- 카드 메타 텍스트: 0.95rem, weight 400

**컬러 토큰 (핵심)**
| 역할 | 색상 |
|------|------|
| 메인 페이지 배경 | #E8E6DD |
| 카드 배경 | #DDD9CD |
| 카드 hover 배경 | #D5D1C5 |
| 메인 텍스트 | #0A0A0A |
| 보조 텍스트 | #5A5A5A |
| 포인트(accent) | #C95652 |
| 그래픽 보조색 | #D4A84F |
| 메뉴 버튼 hover 배경 | #D0CCC0 |
| 성공 배너 | bg #E8EDD8 / border #BCC6A2 / text #4E6440 |
| 에러 배너 | bg #F3E4DF / border #D9B8B1 / text #A44949 |

**형태/간격**
- 메인 페이지 외곽 여백: px-12, py-16
- 콘텐츠 최대 폭: max-w-7xl
- 카드 반경: rounded-3xl
- 썸네일 반경: rounded-2xl
- 그리드: 모바일 1열 / md 2열 / lg 3열, gap-8

**그래픽 컴포넌트**
- CoverGraphic 컴포넌트 사용
- 썸네일 미등록 플레이리스트 카드 및 New playlist 카드에 SVG 패턴 그래픽 표시
- SVG 배경은 #E8E6DD, 분할 원형 포인트는 #C95652 및 #D4A84F

**인터랙션**
- 카드 hover: 배경색 전환 중심(duration 200ms)
- 메뉴 버튼: 원형 hover 배경 강조
- 모달: 전체 오버레이 + 클릭 시 닫힘 유지

#### 페이지별 상태

**PlaylistListPage (메인)**
- Figma 기반 테마가 반영된 상태
- 구성: 헤더(타이틀/서브타이틀) + 플레이리스트 카드 그리드 + New playlist 카드 + 추가/수정 모달
- 카드 공통 구조: 제목/메타 + 점 메뉴 + 정사각 썸네일

**PlaylistDetailPage (상세)**
- 기능 확장 상태: 링크 유형 라벨(YouTube/Apple Music/Soundcloud/link), 플랫폼 필터(모두 보기 포함), 표시 목록 기준 복사
- iTunes 추가 곡의 장르 라벨 표기, 플레이리스트 장르 비율 도넛 차트가 구현된 상태
- 기존 곡 대상 장르/앨범커버 자동 백필 로직이 적용된 상태
- 데스크톱 곡 목록에서 앨범커버 표시 및 커버 미존재 시 도형 플레이스홀더 표시
- 장르 텍스트 원칙: 아티스트와 혼동되지 않도록 보조 시각 계층(작은 크기/중간 톤/라벨 배경)을 사용하며, 데스크톱은 원문 길이를 최대한 보존
- 시각 스타일은 기존 테마와 메인 신규 테마가 혼재된 상태
